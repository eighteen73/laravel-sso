# Contributing

Thank you for considering contributing to the Eighteen73 SSO package.

**Important Notice:** This is an opinionated, internal project designed for the specific needs of eighteen73. While we welcome feedback and pull requests, please be aware that we may decline changes that diverge from our internal architectural preferences or requirements.

## Pull Requests

1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the Laravel coding standards (run `vendor/bin/pint`).
5. Run static analysis (run `vendor/bin/phpstan analyze`).
6. Issue that pull request.

## Testing

Run the tests with Pest:

```bash
vendor/bin/pest
```

## Security Vulnerabilities

If you discover a security vulnerability within this package, please send an e-mail to security@eighteen73.co.uk. All security vulnerabilities will be promptly addressed.
